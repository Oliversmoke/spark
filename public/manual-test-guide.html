0. Before you start

Need: MongoDB connected, AUTH_SECRET set, app running.

  1. Run npm run dev and confirm no startup errors.
  2. Visit /api/health — response shows db: "ok".
  3. Use a fresh test email you have not signed up with before (e.g. you+test1@example.com).
  4. Optional: open DevTools → Network to watch API calls as you test.

1. Landing & auth

/ /signup /login

1a — Landing page (logged out)

  1. Open the home page. You see ComeBack.ai branding and a call to get started.
  2. Click through to Sign up.

1b — Sign up

  1. Enter name, email, password (8+ chars), confirm password. Submit.
  2. You land in the app (usually Chat). No red error toast.
  3. Try signing up again with the same email — you get a clear error (duplicate account).
  • Expect: Toast errors use theme styling, not harsh red blocks.

1c — Log out & log back in

  1. Go to Settings → Log out. You return to login.
  2. Visit /today while logged out — redirected to login.
  3. Log in with your test account. You return to the app.
  4. Wrong password shows an error toast, not a crash.

2. Onboarding

First visit after signup (or reset onboarding in DB for retest).

  1. A 3-step tour appears: set goal → log task → understand recovery.
  2. Step through all 3. Links open Today, goal creation, or Chat.
  3. Finish the tour. It does not show again on refresh.

3. Create a goal — templates

/goals/new

  1. Open New goal. Template gallery shows 4 paths: Software Engineering, Gym Bulk, Language Learning, Reading Habit.
  2. Pick Reading Habit (or any template). Preview shows phases and tasks.
  3. Confirm / create the goal. Success toast or redirect to plan.
  4. Try creating a second active goal — blocked on free tier with upgrade message.
  • Expect: Free tier = 1 active goal. Archive or complete first goal before adding another (unless Pro).

4. Create a goal — wizard & chat

/goals/new /chat

Archive your current goal first (Goal detail page) or use a new test account for this section.

4a — Form wizard

  1. On /goals/new, switch to the form/wizard (non-template) path if available.
  2. Describe a goal (e.g. “Learn React in 3 months”). Generate preview.
  3. Review the path preview (phases, checkpoints, tasks) before saving.
  4. Save the plan. Goal appears in your list.

4b — Chat goal creation

  1. In Chat, send: “I want to learn software engineering” or similar goal phrase.
  2. Coach generates a path (requires valid GEMINI_API_KEY) or shows retry message if AI fails.
  3. Chat shows plan-created message with link to view plan.

5. View your plan

/goals/[id] — open from Progress or goal list

  1. Open your active goal’s detail page.
  2. See full path: phases → checkpoints → tasks with durations.
  3. Checkpoint progress bar reflects logs so far.
  4. If in recovery mode, recovery plan card appears here too.

6. Today — daily logging

/today

  1. Today lists tasks due today, grouped by goal.
  2. Tap Done on a task — optimistic UI, then confirms. XP toast if earned.
  3. Tap Partial on another task — stores partial status (+5 XP).
  4. Tap Skip on a task — stored as skipped, no full XP.
  5. Checkpoint progress updates without full page reload.
  6. 14-day history section shows recent logs with dates and statuses.
  7. Gamification panel shows XP bar, level, and streak state.
  8. Refresh button reloads tasks and history.
  • Expect: First log earns First Log badge.

7. Chat coach

/chat

  1. Chat loads prior messages (including plan creation if any).
  2. Send “I finished reading today” or “done for today” — tasks log via chat.
  3. Success toast shows XP / badges when applicable.
  4. Send a general question — coach replies helpfully (not only logging).
  5. Recovery banner appears when a goal is in recovery mode.
  6. Weekly review banner appears when a review is pending.

8. Progress & gamification

/progress

  1. XP total and level title match what you earned from logging.
  2. Streak shows state: active, at-risk, broken, or recovering.
  3. Badge grid shows earned badges (First Log, etc.). Locked badges are visible.
  4. Per-goal cards show checkpoint progress and recovery status.
  5. Activity history (14 days) matches Today page.

9. Recovery coach

Core differentiator — miss → plan → re-engage

In development only: Goal detail page may have “Simulate miss” for testing. In production, recovery triggers after grace period without logs.
  1. Trigger recovery (simulate in dev, or wait / manually backdate logs in DB).
  2. Recovery banner on Today and Chat with empathetic copy — no shaming.
  3. Recovery plan shows minimum viable day and adjusted tasks.
  4. Accept recovery plan — tasks/deadlines update; path version increments.
  5. Adjust in chat — send why you missed; coach regenerates plan.
  6. Log within 72h of miss — eligible for Recovery Hero badge.

10. Weekly review

/progress and /chat

  1. On Progress, pick a goal and click generate weekly review.
  2. Review references your actual log data from the past week.
  3. Suggested adjustments (add/remove/resize tasks) are readable.
  4. Accept adjustments — active path updates, history preserved via pathVersion.
  5. Dismiss review — no path changes.
  6. Same review accessible from Chat banner.

11. Settings & export

/settings

  1. Settings shows your plan tier (free / pro).
  2. Change reminder time — saves with success toast.
  3. Set timezone and weekly review day — persists after refresh.
  4. Configure quiet hours (start / end).
  5. Toggle push notifications (requires VAPID keys + browser permission).
  6. Export data downloads comeback-ai-export.json with goals and logs.
  7. Log out from Settings works.

12. PWA & offline

  1. Production build (npm run build && npm start): install prompt or “Add to Home Screen” works.
  2. Installed app opens to Today with standalone display.
  3. Load Today once online, then go offline — cached tasks/history still visible.
  4. Offline banner explains you’re viewing cached data.
  5. Push reminder fires at configured time (needs VAPID + cron on Vercel).
Service worker is disabled in dev mode. Test PWA with a production build.

13. Pricing & Pro

/pricing

  1. Pricing page compares Free vs Pro features.
  2. Upgrade button opens Stripe checkout (or “not configured” if keys missing).
  3. After successful checkout (test mode), Settings shows Pro and unlimited goals.

14. Admin (optional)

/admin — your email must be in ADMIN_EMAILS

  1. Non-admin account gets forbidden / redirect on /admin.
  2. Admin sees user count, active goals, logs, recovery funnel metrics.

15. Edge cases

  1. Mobile width (375px): bottom nav works — Chat, Today, Progress, Settings.
  2. Keyboard: Tab through forms and chat input; focus visible.
  3. Invalid AI output (if Gemini returns bad JSON) — user sees retry message, not a crash.
  4. AI rate limit hit — Today quick-log still works; chat explains limit.
  5. 404 page renders for unknown routes.
  6. Full loop complete: goal → log daily → miss → recover → weekly review → export.
